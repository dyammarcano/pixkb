## Findings considered and rejected

- **Ingest source-adapter "duplication"**: the nine `Name()/Fetch()` adapters look parallel
  but their segmentation is genuinely format-specific and they already share the real helpers
  (`slugify`/`firstLine`/`detectMarkdownLang`). Forcing a shared template would couple
  unrelated formats — restraint lens, not a rule-of-three violation.
- **`cmd/pixkb/commands.go` "god file"**: 507 LOC ≈ 3× the package median, not an order of
  magnitude, and cohesive CLI wiring — not a god file.
- **RRF fusion core "coverage gap"**: `internal/query/hybrid.go` is honestly tested with fakes
  (fusion order, title-boost, vector-floor, explain/hit parallelism). No gap.
- **HQL→SQL / Postgres query builders "injection"**: fully parameterized — columns
  whitelisted, operators allow-listed, values positional `$N`. Notably disciplined; no finding.
- **Per-concept multi-write / cross-system non-atomicity as a *correctness* bug**: explicitly
  documented as intentional (`runner.go:36-42`, bundle authoritative + index rebuildable).
  Settled by decision. (It remains a *throughput* item — PERF-01 above.)
